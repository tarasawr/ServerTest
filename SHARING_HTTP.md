# Sharing HTTP API (`projects.js`)

Краткое описание REST-эндпоинтов реестра шаринга проектов. Реализация: [`projects.js`](./projects.js), подключение из [`server.js`](./server.js) через `handleRequest`.

## Общее

- **Базовый URL**: тот же хост и порт, что у HTTP-сервера (по умолчанию `http://localhost:3000`).
- Заголовок для тел JSON: `Content-Type: application/json`.
- Ошибки: JSON `{ "error": "message" }` с соответствующим HTTP-статусом.
- Данные хранятся **в памяти** процесса Node (после перезапуска сервера очищаются).

## Эндпоинты

### `POST /projects`

Регистрация проекта в реестре. Повтор с тем же `projectId` обновляет XML (если передан), дату синхронизации и возвращает успех — **идемпотентно**.

**Тело:**

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `projectId` | да | Строковый идентификатор проекта |
| `ownerUserId` | да | ID владельца |
| `ownerName` | нет | Имя владельца (по умолчанию `Unknown`) |
| `projectXml` | нет | XML снапшот проекта |

**Успех (200):** `{ "ok": true, "projectId": "...", "shareUrl": "/projects/{projectId}" }`  
`shareUrl` — **относительный** путь; полная ссылка = база + `shareUrl`.

**Ошибки:** `400` — нет `projectId` или `ownerUserId`.

---

### `GET /projects/user/:uid`

Список проектов, в которых участвует пользователь `:uid` (URL-декодирование применяется к сегменту).

**Успех (200):** `{ "projects": [ { "projectId", "ownerName", "role", "lastSyncDate" }, ... ] }`

---

### `POST /projects/:id/join`

Добавить пользователя в проект. Новым участникам выдаётся роль `can_view`. Уже существующий пользователь получает свою текущую роль.

**Тело:**

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `userId` | да | ID пользователя |
| `name` | нет | Отображаемое имя |
| `avatarUrl` | нет | URL аватара |
| `isGuest` | нет | boolean |

**Успех (200):**

- новый участник: `{ "ok": true, "role": "can_view" }`
- уже в проекте: `{ "ok": true, "role": "<роль>", "alreadyMember": true }`

**Ошибки:** `404` — проект не найден; `400` — нет `userId`.

---

### `GET /projects/:id`

Метаданные проекта (без полного `projectXml` в ответе).

**Успех (200):** `{ "projectId", "ownerUserId", "ownerName", "lastSyncDate", "userCount" }`

**Ошибки:** `404` — проект не найден.

---

### `GET /projects/:id/users`

Список всех участников.

**Успех (200):** `{ "users": [ { "userId", "name", "avatarUrl", "role", "isGuest" }, ... ] }`

**Ошибки:** `404` — проект не найден.

---

### `GET /projects/:id/users/:uid/role`

Роль конкретного участника (`:uid` URL-декодируется).

**Успех (200):** `{ "userId", "role" }`  
Роли: `owner`, `can_edit`, `can_view`.

**Ошибки:** `404` — проект не найден или пользователь не в проекте.

---

### `PUT /projects/:id/users/:uid/role`

Смена роли участника. Нельзя таким образом «назначить» владельца — только `can_view` или `can_edit`.

**Тело:** `{ "role": "can_view" | "can_edit" }`

**Успех (200):** `{ "ok": true, "role": "..." }`

**Ошибки:** `400` — неверная роль; `404` — проект или пользователь не найден.

---

### `DELETE /projects/:id/users/:uid`

Удалить участника из проекта. **Только владелец.**

**Тело:** `{ "ownerUserId": "<id владельца>" }` — должен совпадать с `ownerUserId` проекта.

**Успех (200):** `{ "ok": true }`

**Ошибки:** `403` — не владелец; `400` — попытка удалить владельца; `404` — проект/пользователь не найден.

---

### `PUT /projects/:id/sync`

Обновить `lastSyncDate` у проекта (тело может быть пустым `{}`).

**Успех (200):** `{ "ok": true, "lastSyncDate": "<ISO8601>" }`

**Ошибки:** `404` — проект не найден.

---

### `PUT /sessions/:inviteCode/project`

Привязать **активную** WebSocket-сессию к зарегистрированному проекту. `inviteCode` — код приглашения сессии на этом же сервере.

**Тело:** `{ "projectId": "..." }`

**Успех (200):** `{ "ok": true }` — у сессии выставляется `projectId`; дальнейшие обновления XML через `update_state` могут синхронизироваться в реестр через `onXmlUpdated`.

**Ошибки:** `400` — нет `projectId`; `404` — проект не найден **или** сессии с таким `inviteCode` нет (частый случай при опечатке или другом инстансе сервера).

---

## Связь с Unity

Клиент Unity использует константу `MultiplayerConnection.HttpUrl` и класс `MultiplayerSharingApi` (`POST /projects`, `POST /projects/{id}/join`, `PUT /sessions/{invite}/project`) — те же контракты, что выше.
